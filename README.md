## `solver_solid3D.m`

3D FE solver using 4-node tetrahedral elements (T4). Assembles sparse K via triplets, applies Dirichlet BCs with RHS correction for non-zero prescribed displacements, solves K·u=F, computes strains, stresses, and principal values.

**Signature:**
```matlab
[U, epsi, epsi1, epsi3, sigma, sigma1, sigma3] = solver_solid3D(nodes, elements, loads, bcs)
```

**Input formats:**
```
nodes    = [node_nr, x, y, z]
elements = [elem_nr, n1, n2, n3, n4, E, nu]
loads    = [node_nr, Fx, Fy, Fz]
bcs      = [node_nr, ux, uy, uz]    % NaN = free DOF
```

**Output:**
```
U      : nodal displacement vector [3*n_nodes x 1]
epsi   : element strains  [n_elem x 7], col 1 = elem nr, cols 2-7 = [ex ey ez gyz gxz gxy]
sigma  : element stresses [n_elem x 7], same layout
epsi1  : max principal strain  per element [n_elem x 1]
epsi3  : min principal strain  per element [n_elem x 1]
sigma1 : max principal stress per element [n_elem x 1]
sigma3 : min principal stress per element [n_elem x 1]
```

**Implementation notes:**
- B-matrix via Jacobian inverse approach (Kochmann §16.2), not hardcoded a/b/c coefficients
- K assembled as sparse via triplets — no dense matrix allocated
- BC application entirely on triplets before sparse build — no in-place sparse modification
- Non-zero Dirichlet BCs: RHS correction `F = F - K(:,d)*r` before row/column zeroing
- Voigt ordering: `[εx εy εz γyz γxz γxy]`

## Notes

- Ansys uses SOLID187 (10-node quadratic tet) vs your T4 (4-node linear tet) — some difference in results is expected and should be discussed in the report
- Task 3 uses prescribed displacement of −1 mm at the greater trochanter, not a force — this is why the non-zero BC method is required
- `nu = 0.3` is hardcoded in `WriteFemurInputFile_rev2` for all elements regardless of material mapping method
